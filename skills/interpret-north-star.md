# Skill: Interpret North Star

Use this skill when the user provides a north star document, presentation, or diagram
and wants to extract structured product intent from it.

## What to produce

Read the provided material and break it into the following sections. Ask for the
material first if it has not been provided.

**Problem** — What specific pain or gap does this address? One or two sentences,
no jargon.

**Goals** — What must be true for this to be considered successful? Measurable where
possible. Bullet list.

**Non-goals** — What is explicitly out of scope? Naming non-goals prevents scope creep
and misaligned engineering effort.

**User stories** — Who is the user and what do they need to do? Format:
`As a <user type>, I want to <action> so that <outcome>.`

**Success metrics** — How will you know it worked? Include leading and lagging
indicators. Tie to goals.

**Open questions** — Assumptions that need validation, decisions not yet made,
stakeholders to consult.

## Rules

- Do not invent goals the source material doesn't support.
- If the source is ambiguous on a goal, list it as an open question.
- Ask one clarifying question before starting if the material is very sparse.
